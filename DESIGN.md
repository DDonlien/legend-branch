# LegendBranch 视觉规范

## 视觉主题

- 方向：Quiet Atlas / 沉静图谱风。
- 关键词：极简信息制图、数字档案、现代博物馆、历史地图册、低调系统感。
- 核心气质：主图沉静、清晰、适合长时间阅读；Trace 时才明显发光。
- 应避免：羊皮纸、卷轴、毛笔、古风纹理；全屏玻璃折射；持续高饱和 Acid Design；过强的科幻设备感。

## 视觉语言分工

- 主界面：极简信息制图，保证高密度图谱可读性。
- 浮层：轻微 Liquid Glass，用于搜索栏、节点详情和 Trace 控制。
- 控件：少量 Nothing 式系统感，可用于时间刻度、节点编号和状态标签。
- Trace 高亮：少量 Acid Design，用作分析状态的视觉高潮。

## 色彩

以下 token 是 P1 初始方向，开发时可微调，但要保持低饱和背景与醒目 Trace 的对比关系。

--bg-panel: rgba(22, 25, 30, 0.72);

## 字体

- 正文：优先使用清晰、中性的无衬线字体。
- 等宽字体：用于时间刻度、节点编号、可信度标签和 `TRACE MODE` 状态。
- 原则：字体服务于信息层级，不用装饰性古风字体承载正文。

## 布局

- 页面主体由顶部工具栏和大面积图谱画布组成。
- 顶部搜索 / 筛选栏占一整行，视觉低调，并支持隐藏。
- 画布支持横向和纵向滚动。
- 横轴表示时间。
- 纵轴由文化、国家或政权区块构成；区块包含若干角色行，行高根据关系组织动态变化。
- 区块可以随历史演进合并或分叉，使用低透明度背景和柔和边界表达。

## 节点与连线

### 节点

- 默认节点接近 Git commit 点，不做复杂卡片。
- P1 节点类型：
  - `●` 神话 / 民俗来源
  - `◆` 古典文学版本
  - `■` 现代作品版本

### 连线

- `confirmed`：实线。
- `probable`：虚线。
- `speculative`：点线。
- 默认线条细且克制；Trace 路径使用高亮色和轻微发光。

## Trace 状态

- 相关节点：`100%` 不透明度。
- 相关路径：高亮并轻微发光。
- 无关节点：约 `10%` 不透明度。
- 无关路径：约 `5%` 不透明度。
- 顶部显示低调状态条，例如 `TRACE MODE · 孙悟空`。

## 节点详情浮层

- 点击节点后显示 popup，不切换页面，不使用全屏 modal。
- popup 像半透明档案卡或博物馆展签，玻璃效果轻微。
- P1 内容：节点名、英文名或原名、时代、文化区域、类型、一句话简介、来源 / 影响关系、外链索引。

## 动效

- 动效应服务于阅读方向、区块层次和 Trace 状态切换。
- 默认界面避免持续动画和强烈发光。
- Trace 进入、退出和 popup 开合使用短促、克制的过渡。
- 支持 `prefers-reduced-motion`。

## 可访问性

- 可信度不能只依赖颜色区分，必须同时使用线型。
- 节点类型不能只依赖颜色区分，必须同时使用形状。
- 浮层、工具栏和 Trace 清除操作需要可见焦点态与键盘访问路径。
- 文字与背景保持足够对比度。
